## script标签引用位置规范

`<script>` 放的位置，会直接影响：

- 页面加载速度
- DOM 是否能获取到
- 是否阻塞渲染
- 用户体验

### 最传统写法（早期）

以前很多人这样写：

```html
<head>
    <script src="app.js"></script>
</head>
```

但这是不推荐的。

### 为什么不推荐？

因为：

```txt
script 会阻塞 HTML 解析
```

浏览器执行流程：

```txt
下载 HTML
↓
遇到 script
↓
停止解析 HTML
↓
下载 js
↓
执行 js
↓
继续解析 HTML
```

结果：

- 页面白屏更久
- 渲染变慢
- 性能差

### 传统规范（经典推荐）

把 script 放到 `body` 最后。

#### 推荐写法

```html
<body>

<div id="app"></div>

<script src="app.js"></script>
</body>
```

为什么？

因为：

```txt
HTML 先加载完
再执行 JS
```

好处：

- 页面先显示
- JS 能拿到 DOM
- 不容易报错

举例

如果：

```js
document.getElementById('app')
```

但 DOM 还没解析到：

```html
<div id="app"></div>
```

就会：

```txt
null
```

所以以前：

```txt
script 放 body 最后
```

几乎是行业默认规范。

### 现代规范（推荐）

现在更推荐：

```html
<head>
    <script src="app.js" defer></script>
</head>
```

### defer 是什么？

`defer`：

```txt
延迟执行
```

浏览器行为：

```txt
HTML 继续解析
JS 后台下载
HTML 解析完毕
再执行 JS
```

### defer 的优势

#### 不阻塞页面

页面更快。

#### DOM 一定存在

因为：

```txt
HTML 已经解析完成
```

#### 适合大型项目

Vue / React / Vite 基本都这样。

### 现代项目真实写法

例如 Vue/Vite：

```html
<head>
  <script type="module" src="/src/main.js"></script>
</head>
```

这里：

```txt
type="module"
```

模块脚本默认就带 defer 特性。

所以：

```txt
不会阻塞 HTML
```

### script 标签几个重要位置对比

| 写法             | 是否推荐 | 原因         |
| ---------------- | -------- | ------------ |
| head 普通 script | ❌        | 阻塞解析     |
| body 最后        | ✅        | 传统最佳实践 |
| head + defer     | ✅✅       | 现代最佳实践 |
| head + async     | 特殊情况 | 不保证顺序   |

### async 和 defer 区别（面试高频）

#### async

```html
<script async src="a.js"></script>
```

特点：

```txt
下载完立即执行
```

问题：

```txt
执行顺序不可控
```

适合：

- 广告
- 统计
- 第三方脚本

例如：

- Google Analytics
- 广告 SDK

#### defer

```html
<script defer src="a.js"></script>
```

特点：

```txt
HTML 解析完成后执行
并且按顺序执行
```

适合：

```txt
业务代码
```

### 现在真正行业规范

### 原生小页面

推荐：

```html
<body>
    ...
    <script src="app.js"></script>
</body>
```

简单直接。

### 现代工程化项目

推荐：

```html
<script type="module" src="main.js"></script>
```

或者：

```html
<script defer src="main.js"></script>
```

### 真正核心原则

记住一句：

> JS 不要阻塞 HTML 渲染。

所以：

```txt
能 defer 就 defer
```

### 你以后写原生 JS 最安全的方式

直接记这个：

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Demo</title>
</head>
<body>

<div id="app"></div>

<script src="app.js"></script>
</body>
</html>
```

或者现代版：

```html
<script defer src="app.js"></script>
```

这两个都是标准规范。
