## 为什么vue 总用 template结构而不像h5一样的**html-> head->body**结构

 **Vue3** 和 **传统原生 HTML** 的核心区别。

### 为什么 Vue3 必须用 `<template>` + `<div>`，而不像原生 H5 那样写？

#### 1. 原生 H5 是 “完整网页”，Vue 是 “组件碎片”

**原生 H5**：

```html
<html>
  <head>...</head>
  <body>
    <div>内容</div>  <!-- 直接写在 body 里 -->
  </body>
</html>
```

它是**一整个完整的网页**，有头有身体。

**Vue 组件**：

它**不是一个完整网页**，只是网页里的**一小块功能（组件）**。

比如：一个按钮、一个弹窗、一个列表项。

Vue 需要一个 **“容器”** 来装这一小块代码 → 这个容器就是 `<template>`。

#### 2. Vue 模板必须有 “根节点”

Vue 有一个硬性规则：

**`<template>` 内部，必须有且只有一个根标签包裹所有内容。**

不能写成这样（会报错）：

```vue
<template>
  <!-- ❌ 错误：两个根节点并列 -->
  <div>1</div>
  <div>2</div>
</template>
```

必须包一层：

```vue
<template>
  <!-- ✅ 正确：一个根节点包住所有 -->
  <div>
    <div>1</div>
    <div>2</div>
  </div>
</template>
```

大家习惯用 `<div class="box">` 做这个**根容器**。

#### 3. `<template>` 不是真实 DOM，它是 “模板语法”

重点来了：

- `<template>` **不会渲染到页面上**
- 浏览器不认识它，Vue 认识它
- 它只是用来**写布局的 “草稿纸”**

最终渲染到页面的，只有里面的 `<div>`。

#### 4. 为什么不像 H5 那样用 `<body>`？

因为：

- **一个页面只有一个 `<body>`**
- Vue 是组件化开发，一个页面可能有 10～100 个组件
- 每个组件不可能都带 `<html> <head> <body>`

所以 Vue 创造了一套**组件结构**：

```vue
<template>  <!-- 结构（HTML） -->
  <div>内容</div>
</template>

<script setup>  <!-- 逻辑（JS/TS） -->
  // 代码
</script>

<style scoped>  /* 样式（CSS） */
  /* 代码 */
</style>
```

### 一句话总结

**原生 H5 = 完整网页**

**Vue 组件 = 网页碎片**

- `<template>`：装碎片的**容器**
- 里面的 `<div>`：碎片的**根节点**（必须有）

### uni-app 对比 Vue:



|    框架     | 最外层根标签推荐 |              原因              |
| :---------: | :--------------: | :----------------------------: |
|  **Vue3**   |     `<div>`      |  运行在浏览器，支持原生 HTML   |
| **uni-app** |     `<view>`     | 运行在小程序 / App，不支持 div |

**结构完全一样，只是把 div 换成了 view**：

Vue3：

```vue
<template>
  <div>内容</div>
</template>
```

uni-app：

```vue
<template>
  <view>内容</view>
</template>
```

------

### 超级浓缩结论

1. `<template>` 是 Vue 规定的**组件外壳**，不渲染
2. 里面必须包一个**根元素**
3. Vue 用 `<div>`，uni-app 用 `<view>`
4. 因为 Vue 是**组件碎片**，不是完整网页
