## 事情原因

> 使用**@**符号发现  `api.` 没有自动提示了

```js
import api from '@/api'
const res = await api.auth.login(loginForm.value.username, loginForm.value.password)
```





## 配置 ` vite.config.js`

```js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import AutoImport from 'unplugin-auto-import/vite'
import Components from 'unplugin-vue-components/vite'
import { ElementPlusResolver } from 'unplugin-vue-components/resolvers'

// 👉 就缺这一行！
import path from 'path'   //一定要导入这个

export default defineConfig({
  plugins: [
    vue(),
    AutoImport({ resolvers: [ElementPlusResolver()] }),
    Components({ resolvers: [ElementPlusResolver()] }),
  ],

  // ======================
  // 关键：路径别名配置
  // ======================  
  resolve: { 
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  }
    
    
})
```

### 讲解

```js
resolve: { alias: { '@': path.resolve(__dirname, 'src') } } //这是什么意思
```

这段是 Vite 的“路径别名（alias）”配置。

意思是：

```
@
```

代表：

```
src
```

不配置 alias 时

你可能要这样写：

```js
import api from '../../../api'
```

层级深了会非常难看。

配置 alias 后

```js
resolve: {
  alias: {
    '@': path.resolve(__dirname, 'src')
  }
}
```

表示：

```js
@  =>  项目根目录/src
```

所以：

```js
import api from '@/api'
```

实际等价于：

```js
import api from '/src/api'
```

逐行解释

**resolve**

Vite 的模块解析配置。

**alias**

定义“别名”。

**'@'**

你自己定义的名字。

可以改成：

```js
'#'
'~'
'root'
```

都行。

只是 Vue/Vite 社区默认用 `@` 表示 `src`。

path.resolve(__dirname, 'src')

意思：

```js
项目根目录 + src
```

例如：

项目：

```js
D:/project/my-vue-app
```

那么：

```js
path.resolve(__dirname, 'src')
```

得到：

```js
D:/project/my-vue-app/src
```

为什么 WebStorm 不识别？

因为：

```
vite.config.ts
```

只是：

Vite运行时配置

IDE 不一定看它。

所以：

还必须告诉 TypeScript：

```js
{
  "compilerOptions": {
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

这样：

- Vite 能运行
- WebStorm 能补全
- TypeScript 能识别

三者才统一。





## jsonconfig.json

[文档](https://vscode.js.cn/docs/languages/jsconfig)

### 这个配置是干嘛用的？

**不是给项目运行用的，是给 WebStorm / VSCode 用的**

作用只有 2 个：

1. **让 @ 路径能跳转**
2. **让 api.auth.login 出现代码提示**

### **代码**

```js
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "dist"]
}
```

**加注释的代码**

```js
{
  "compilerOptions": {
    // 基础目录：. 代表从项目根目录开始解析路径
    "baseUrl": ".",
    // 路径别名映射：告诉编辑器 @/xxx 对应 src/xxx
    "paths": {
      // @/* 匹配所有以 @/ 开头的路径 → 指向 src/ 下的对应文件
      "@/*": ["src/*"]
    }
  },
  // 【项目包含范围】：编辑器只分析 src 下的所有 JS/Vue 文件
  "include": ["src/**/*"],
  // 【排除范围】：不分析这些文件夹，提升编辑器速度
  "exclude": [
    "node_modules",  // 依赖包
    "dist"           // 打包产物
  ]
}
```

### 每一句作用（超清晰版）

#### 1. `"baseUrl": "."`

- 意思：**所有路径都从项目根目录开始算**
- 没有这个，`paths` 别名不生效

#### 2. `"@/*": ["src/*"]`

- 核心作用：

  

  ```
  @/api
  ```

   → 对应 

  ```
  src/api
  ```

  

  ```
  @/utils
  ```

   → 对应 

  ```
  src/utils
  ```

- 让 WebStorm/VSCode **认识 @ 符号**

#### 3. `"include": ["src/**/*"]`

- 让编辑器只检查 `src` 里的代码
- 支持所有子目录

#### 4. `"exclude": [...]`

- 跳过不需要分析的目录
- 让 WebStorm 不卡顿
