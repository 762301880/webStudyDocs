# 前端 API 封装规范（企业级）

## 一、核心原则

避免在 Vue 模板 / 组件中直接编写 API 请求逻辑，通过**按业务模块封装 API 方法**的方式，实现接口逻辑的统一管理，解决多模板重复调用相同接口、接口变更维护成本高的问题（类比 PHP Service 封装思想）。

## 二、基础示例：单一接口封装

以企业最常用的 `refreshToken` 接口为例，直观理解封装思路：

### 封装实现（auth.js）

```js
// src/api/auth.js
import request from '../utils/request'

/**
 * 刷新登录令牌
 * @returns {Promise} 请求Promise对象
 */
export function refreshToken() {
    return request.post('/api/auth/refresh', {}, {
        isRefreshToken: true
    })
}
```

### 业务层调用

```js
import { refreshToken } from '@/api/auth'

await refreshToken()
```

### 封装优势

- 接口地址对业务层透明，降低前后端耦合；
- 接口变更时仅需修改封装层，无需改动所有调用处；
- 刷新令牌的逻辑统一收敛，便于维护。

## 三、标准化封装方案（企业级）

### 1. 目录结构规范

按业务模块拆分 API 文件，避免所有请求集中在单个 `api.js` 中：

```
src/
├── api/                // 所有业务接口封装目录
│   ├── auth.js         // 认证模块（登录/刷新令牌/登出）
│   ├── user.js         // 用户模块（信息查询/修改）
│   ├── article.js      // 文章模块
│   └── order.js        // 订单模块
│
├── utils/
│   └── request.js      // axios 实例封装（拦截器/Token/刷新逻辑）
```

### 2. 核心文件实现

#### （1）utils/request.js（axios 实例封装）

封装 axios 基础配置、请求 / 响应拦截器、Token 管理、刷新令牌逻辑，是所有 API 请求的基础：

```javascript
import axios from 'axios'
import { ElMessage } from 'element-plus'
import TokenUtil from '../utils/token'
import router from '@/router' // 补充引入路由（原代码遗漏）

// 创建 axios 实例
const service = axios.create({
    baseURL: import.meta.env.VITE_API_URL, // 环境变量配置接口前缀
    timeout: 10000 // 请求超时时间
})

/**
 * 状态标识：是否正在刷新 Token
 */
let isRefreshing = false

/**
 * 等待队列：存储 Token 刷新期间待执行的请求
 */
let requests = []

/**
 * 请求拦截器：添加 Token 到请求头
 */
service.interceptors.request.use(
    (config) => {
        const token = TokenUtil.getAccessToken()
        if (token) {
            config.headers.Authorization = `Bearer ${token}`
        }
        return config
    },
    (error) => Promise.reject(error)
)

/**
 * 响应拦截器：统一处理错误、Token 刷新
 */
service.interceptors.response.use(
    // 响应成功：直接返回业务数据
    (response) => response.data,
    // 响应失败：统一处理异常
    async (error) => {
        const { response, config } = error

        // 网络异常（无响应）
        if (!response) {
            ElMessage.error('网络异常，请检查网络连接')
            return Promise.reject(error)
        }

        // Token 过期（401 状态码）
        if (response.data.code === 401) {
            // 刷新 Token 接口自身返回 401 → 令牌彻底过期，需重新登录
            if (config.url.includes('/api/auth/refresh')) {
                localStorage.clear()
                ElMessage.error('登录已过期，请重新登录')
                router.push('/login')
                return Promise.reject(error)
            }

            // 若正在刷新 Token，将当前请求加入等待队列
            if (isRefreshing) {
                return new Promise((resolve) => {
                    requests.push((newToken) => {
                        config.headers.Authorization = `Bearer ${newToken}`
                        resolve(service(config))
                    })
                })
            }

            // 开始刷新 Token
            isRefreshing = true
            try {
                // 调用刷新 Token 接口
                const res = await axios.post(
                    `${import.meta.env.VITE_API_URL}/api/auth/refresh`,
                    { refresh_token: TokenUtil.getRefreshToken() }
                )
                const newToken = res.data.data.access_token
                
                // 保存新 Token 并更新请求头
                TokenUtil.setAccessToken(newToken)
                
                // 执行等待队列中的所有请求
                requests.forEach((cb) => cb(newToken))
                requests = []

                // 重试当前失败的请求
                return service(config)
            } catch (e) {
                // 刷新 Token 失败 → 登出并跳转登录页
                localStorage.clear()
                ElMessage.error('登录已过期，请重新登录')
                router.push('/login')
                return Promise.reject(e)
            } finally {
                // 结束刷新状态
                isRefreshing = false
            }
        }

        // 其他业务错误
        ElMessage.error(response.data.msg || '请求失败')
        return Promise.reject(error)
    }
)

export default service
```

#### （2）api/auth.js（认证模块）

统一管理登录、刷新令牌、登出等认证相关接口：

```js
// src/api/auth.js
import request from '@/utils/request'

/**
 * 用户登录
 * @param {Object} data - 登录参数（账号/密码等）
 * @returns {Promise} 请求Promise对象
 */
export function login(data) {
    return request.post('/auth/login', data)
}

/**
 * 刷新登录令牌
 * @param {Object} data - 刷新参数（refresh_token）
 * @returns {Promise} 请求Promise对象
 */
export function refreshToken(data) {
    return request.post('/auth/refresh', data, { isRefreshToken: true })
}

/**
 * 用户登出
 * @returns {Promise} 请求Promise对象
 */
export function logout() {
    return request.post('/auth/logout')
}
```

#### （3）api/user.js（用户模块）

统一管理用户信息查询、修改等接口：

```js
// src/api/user.js
import request from '@/utils/request'

/**
 * 获取当前用户信息
 * @returns {Promise} 请求Promise对象
 */
export function getUserInfo() {
    return request.get('/user/info')
}

/**
 * 更新用户信息
 * @param {Object} data - 用户修改参数
 * @returns {Promise} 请求Promise对象
 */
export function updateUser(data) {
    return request.put('/user', data)
}
```

### 3. 业务层调用规范

组件 / 页面中仅需导入封装好的 API 方法，无需关注接口地址、Token、拦截器等细节：

```js
// 组件中调用示例
import { login } from '@/api/auth'
import { getUserInfo } from '@/api/user'

// 登录请求
const handleLogin = async (form) => {
    try {
        const res = await login(form)
        // 登录成功逻辑
    } catch (error) {
        // 错误处理
    }
}

// 获取用户信息
const fetchUserInfo = async () => {
    try {
        const res = await getUserInfo()
        // 处理用户信息
    } catch (error) {
        // 错误处理
    }
}
```

## 四、封装的核心价值

1. **降低维护成本**：接口地址 / 逻辑变更时，仅需修改封装层，无需改动所有调用处（例如 `/api/user/info` 改为 `/api/v2/users/profile` 时，仅需修改 `user.js` 中的 `getUserInfo` 方法）；
2. **逻辑统一收敛**：Token 管理、请求头、错误处理等逻辑集中在 `request.js`，避免重复代码；
3. **提高可读性**：业务层仅关注 “做什么”（如 `login()`、`getUserInfo()`），而非 “怎么做”（如拼接 URL、设置请求头）；
4. **便于协作**：按业务模块拆分文件，前端开发人员可分工维护不同模块的接口，降低冲突概率。

## 五、禁止规范

1. 禁止在组件 / 模板中直接使用 `axios.post/get` 等原生方法编写请求；
2. 禁止将所有接口集中在单个 `api.js` 文件中，必须按业务模块拆分；
3. 禁止在业务层处理 Token 刷新、请求头设置等通用逻辑，统一收敛到 `request.js`。

## 补充

###  假如有100个方法 我难道要导入100次吗(推荐用下面那种方案)

最佳方案：模块统一导出 + 整体引入

####  1. 改造 `user.js`（统一导出）

把所有方法打包成**一个对象**导出，而不是散着导出：

```js
// src/api/user.js
import request from '@/utils/request'

// 获取当前用户信息
export const getUserInfo = () => {
  return request.get('/user/info')
}

// 更新用户信息
export const updateUser = (data) => {
  return request.put('/user', data)
}

// 假设有100个方法，就这样一直写下去...
export const getUserList = () => request.get('/user/list')
export const deleteUser = (id) => request.delete(`/user/${id}`)
export const changeStatus = (data) => request.put('/user/status', data)
// ... 96 个更多方法
```

#### 2. 组件里 **只 import 1 次**

```js
// 组件中
import * as userApi from '@/api/user'

// 使用
userApi.getUserInfo()
userApi.updateUser()
userApi.getUserList()
userApi.deleteUser()
```

**不管 100 个、200 个方法，永远只需要 1 行 import！**

### 更高级：API 统一入口（超级推荐）就这种超级推荐

企业项目几乎都这么做，**所有模块只引入一个 API 对象**，结构超级清晰。

真实企业 90% 都用 **第二种：全局挂载 $api**

#### 步骤 1：新建 `src/api/index.js`

```js
// src/api/index.js
import * as auth from './auth'
import * as user from './user'
import * as order from './order'
import * as article from './article'

// 全部挂载到一个 API 对象上
export default {
  auth,
  user,
  order,
  article
}
```

#### 步骤 2：全局挂载（main.js）

```js
import api from '@/api'
app.config.globalProperties.$api = api
```

#### 步骤 3：任意组件里直接用（零 import！）

```js
// 组件中
this.$api.user.getUserInfo()
this.$api.user.updateUser()

this.$api.auth.login()
this.$api.order.getList()
```

**终极效果：**

不管多少接口、多少模块，**0 行 import，直接调用！**

### 真实使用(可行方案)

```js
//index.vue

import  api from '../api'
const res = await api.auth.getUserInfo() //使用
```

####  为什么可以自动识别到index.js之内的

这是 ES Module 的一个规则。

你这里：

```js
import api from '../api'
```

实际上：

浏览器/Vite 会自动补全：

```js
import api from '../api/index.js'
```

因为：

当 import 一个目录时：

```js
import xxx from './目录名'
```

JS 会默认去找：

```js
目录名/index.js
```

所以你的：

```js
import api from '../api'
```

等价于：

```js
import api from '../api/index.js'
```

你的目录结构应该类似：

```js
src
├─ api
│  ├─ index.js
│  └─ auth.js
```

然后流程是：

```js
import api from '../api'
```

↓

自动变成：

```js
import api from '../api/index.js'
```

第二步

index.js 内容：

```js
import * as auth from './auth'

export default {
    auth
}
```

第三步

这里：

```js
import * as auth from './auth'
```

意思是：

把 auth.js 里所有 export 全部收集起来

例如：

auth.js：

```js
export function login() {}

export function logout() {}
```

那么：

```js
auth
```

就会变成：

```js
{
    login,
    logout
}
```

第四步

然后：

```
export default {
    auth
}
```

等于：

```js
export default {
    auth: {
        login,
        logout
    }
}
```

第五步

于是：

```js
import api from '../api'
```

得到的：

```js
api
```

其实就是：

```js
{
    auth: {
        login,
        logout
    }
}
```

所以最终：

```
api.auth.login()
```

就成立了。
