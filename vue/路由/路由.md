## 后台请求路由常量统一封装

### 完整 `@/constants/route.js`

```js
/**
 * 路由路径常量
 * 统一管理所有路由地址，避免硬编码(ps:这个注册的是vue路由表自定义的路由常量封装)
 */
export const ROUTE_PATH = {
  // 基础公共路由
  LOGIN: '/login',                // 登录页
  REGISTER: '/register',          // 注册页
  FORGET_PWD: '/forget-pwd',     // 忘记密码
  HOME: '/',                     // 首页/工作台
  PROFILE: '/profile',           // 个人中心

  // 系统管理模块（后台常用）
  SYSTEM: '/system',             // 系统管理父路由
  SYSTEM_USER: '/system/user',   // 用户管理
  SYSTEM_ROLE: '/system/role',    // 角色管理
  SYSTEM_MENU: '/system/menu',    // 菜单管理
  SYSTEM_DEPT: '/system/dept',    // 部门管理
  SYSTEM_DICT: '/system/dict',    // 字典管理
  SYSTEM_LOG: '/system/log',      // 操作日志

  // 业务示例模块（可根据自身业务增删）
  DASHBOARD: '/dashboard',       // 数据大屏/统计面板
  ORDER: '/order',               // 订单管理
  GOODS: '/goods',               // 商品管理
  NOTICE: '/notice',             // 公告通知

  // 异常页面
  PAGE_401: '/401',              // 无权限
  PAGE_403: '/403',              // 禁止访问
  PAGE_404: '/404',              // 页面不存在
  PAGE_500: '/500'               // 服务器错误
}

/**
 * 路由名称常量（配合命名路由跳转，推荐优先使用）
 * 和路由配置中的 name 一一对应
 */
export const ROUTE_NAME = {
  LOGIN: 'Login',
  REGISTER: 'Register',

  HOME: 'Home',
  PROFILE: 'Profile',

  SYSTEM: 'System',
  SYSTEM_USER: 'SystemUser',

  SYSTEM_MENU: 'SystemMenu',
  SYSTEM_DEPT: 'SystemDept',
  SYSTEM_DICT: 'SystemDict',

  DASHBOARD: 'Dashboard',

  PAGE_401: 'Page401',

```

### 配套使用方式

#### 1. 路由文件 `@/router/index.js` 引用绑定

```js
import { createRouter, createWebHistory } from 'vue-router'
import { ROUTE_PATH, ROUTE_NAME } from '@/constants/route'

const routes = [
  {
    path: ROUTE_PATH.LOGIN,
    name: ROUTE_NAME.LOGIN,
    component: () => import('@/views/login/index.vue')
  },
  {
    path: ROUTE_PATH.HOME,
    name: ROUTE_NAME.HOME,
    component: () => import('@/views/home/index.vue')
  },
  {
    path: ROUTE_PATH.PAGE_404,
    name: ROUTE_NAME.PAGE_404,
    component: () => import('@/views/error/404.vue')
  },
  // 其他路由同理...
  // 兜底404
  { path: '/:pathMatch(.*)*', redirect: ROUTE_PATH.PAGE_404 }
]

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes
})

export default router
```

#### 2. 代码中跳转（两种写法任选）

```js
// 方式1：路径跳转
import { ROUTE_PATH } from '@/constants/route'
router.push(ROUTE_PATH.LOGIN)

// 方式2：命名路由（企业更推荐，path改了也不受影响）
import { ROUTE_NAME } from '@/constants/route'
router.push({ name: ROUTE_NAME.LOGIN })
```

#### 3.axios 拦截器改造示例

```js
import { ROUTE_NAME } from '@/constants/route'

// 原硬编码代码替换
router.push({ name: ROUTE_NAME.LOGIN })
```

### 扩展补充规则（团队规范）

1. 命名规范

   - 路径：全小写，多单词用 `-` 连接（如 `forget-pwd`）
   - 路由 name：大驼峰，和页面 / 路由语义一致

2. 分层原则

   

   父路由统一前缀（如 

   ```
   /system
   ```

   ），子路由继承，方便权限、菜单批量控制；

3. 按需删减

   

   没有的业务模块（如商品、订单）直接删除对应常量即可；

4. 新增路由要求

   

   后续每加一个页面，必须同步在 

   ```
   route.js
   ```

    补充对应常量，禁止再写死字符串。

### 这样封装的好处

>  路由常量统一封装 核心好处（精简概述，可直接用于团队文档 / 注释）

#### 1. **统一维护，改一处全项目生效**

路由路径 / 名称集中管理，修改地址无需全局搜索替换，避免漏改、出错。

#### 2.**杜绝硬编码，提升代码规范性**

全程使用常量变量，不再手写字符串，减少拼写错误，代码可读性更强。

#### 3. **降低协作成本，统一团队标准**

多人开发共用一套路由规则，新人上手直观，路由语义清晰、一目了然。

#### 4. **适配命名路由，兼容性更强**

搭配路由 `name` 使用时，**路径变更不影响业务代码**，适配动态路由、权限控制。

#### 5.**便于扩展与管控**

新增页面、模块、异常页只需在常量文件追加，后期拓展、路由权限、菜单配置更便捷。

#### 5.**统一跳转逻辑，减少重复 BUG**

拦截器、登出、权限拦截等多处跳转逻辑复用常量，行为一致，便于统一优化。
