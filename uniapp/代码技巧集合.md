### 条件编译

条件编译是用特殊的注释作为标记，在编译时根据这些特殊的注释，将注释里面的代码编译到不同平台。

### 使用方法

以 `#ifdef` 或 `#ifndef` 加 `%PLATFORM%` 开头，以 `#endif` 结尾。

- `#ifdef`：if defined 仅在某平台存在
- `#ifndef`：if not defined 除了某平台均存在
- `%PLATFORM%`：平台名称

### sql表设计

### 相关字段的解释

| 字段         | 作用                 | 示例             |
| ------------ | -------------------- | ---------------- |
| app_id       | 标识是哪个App        | `__UNI__1234567` |
| version_name | 给用户看的版本号     | `3.0.1`          |
| version_code | 程序比较大小的版本号 | `111`            |

#### app_id

用于识别应用身份。

相当于：

```shell
身份证号
```

例如：

```shell
用户端App
商家端App
管理端App
```

数据库：

```shell
app_id=__UNI__USER
app_id=__UNI__SHOP
app_id=__UNI__ADMIN
```

UniApp获取：

```javascript
console.log(plus.runtime.appid)
```

例如：

```php
__UNI__F4A1234
```

这个值基本不会变。

#### version_name

用户看到的版本号。

例如：

```bash
我的 -> 关于我们

当前版本：
3.0.1
```

App Store：

```bash
Version 3.0.1
```

Android：

```bash
版本 3.0.1
```

都是这个。

升级过程：

```bash
3.0.0
↓
3.0.1
↓
3.0.2
↓
3.1.0
```

用户能看懂。

#### version_code(版本控制比较用这个字段)

程序自己比较大小。

例如：

```bash
3.0.1
```

对应：

```bash
111
```

下一版：

```bash
3.0.2
```

对应：

```bash
112
```

再下一版：

```bash
3.1.0
```

对应：

```bash
120
```

因为程序不能直接这样比较：

```bash
"3.0.10" > "3.0.2"
```

结果可能不符合预期。

所以一般比较：

```bash
111 < 112
```

最简单。



#### 这个表里怎么用

假设：

```bash
当前安装版本
3.0.0
version_code=110
```

服务器返回：

```json
{
  "name": "我的APP",
  "appid": "__UNI__xxxxxx",
  "versionName": "1.0.0", // 展示版本
  "versionCode": 100      // 数字递增，热更靠这个对比，每次打WGT必须+1
}
```

检测：

```javascript
if(server.version_code > local.version_code){
    // 需要升级
}
```

数据库：

```bash

version_name='3.0.1'

version_code=111
```

显示给用户：

```bash
当前版本：3.0.1
```

代码判断：


#### 如果只有一个App

甚至可以理解成：

```bash
app_id
    ↓
哪个App

version_name
    ↓
给人看的

version_code
    ↓
给程序比较的
```

最简单的例子：

```bash
app_id       = __UNI__F4A1234

version_name = 3.0.1

version_code = 111
```

以后发新版：

```bash
app_id       = __UNI__F4A1234  （不变）

version_name = 3.0.2

version_code = 112
```

真正用于“检测是否需要更新”的核心字段其实是：

```bash
version_code
```

程序只需要比较：

```bash
服务器version_code > 本地version_code
```

就知道是否有新版本。

###  **版本判断最好放在 `App.vue` 的 `onLaunch()` 里。**

原因：

- App启动立即检查
- 所有页面进入前就能拦截
- 强制更新可以直接阻止用户继续使用
- 绝大多数项目都是这么做

```js
// App.vue
export default {
    onLaunch() {
        this.checkVersion()
    }
}
```

如果你担心每次启动都请求接口，可以：

```js
App启动
↓
检查版本
↓
有新版本弹窗
↓
进入首页
```

###  具体业务逻辑 util/version.js

```js
import {
	request
} from "@/utils/request.js"
/**
 * 检查版本更新
 */
export async function checkVersion() {
	// #ifdef H5
	     return
	// #endif

	try {
		const fn = async () => {
			const res = await request('app_version/appVersion', {}, 'GET')
			const {
				code,
				msg,
				data
			} = res

			if (code !== 1 || !data) {
				return
			}

			const serverVersion = Number(data.version_code)
			const localVersion = Number(plus.runtime.versionCode || 0)
			// 读取本地缓存：上次成功热更的最高版本，无数据默认0
			let localWgtVer = Number(uni.getStorageSync('wgt_update_ver') || 0)

			// 完整拦截逻辑：满足任意一条，直接终止，不弹出更新弹窗
			// 1. 原生安装包版本 >= 服务端版本（整包已更新，无需热更）
			// 2. 本地已装过同等/更高版本WGT（本次热更已完成，不再弹）
			if (localVersion >= serverVersion || localWgtVer >= serverVersion) {
				console.log("无需更新：原生版本或本地热更缓存已达标")
				return
			}

			const update_type = data.update_type || "" //更新类型：1-热更新，2-整包更新

			if (update_type == 1) {
				showWgtUpdateModal(data)
			}

			if (update_type == 2) {
				showUpdateModal(data)
			}
		}
		fn()
	} catch (error) {
		console.error('检查更新失败', error)
	}
}

/**
 * WGT热更新弹窗
 */
function showWgtUpdateModal(versionInfo) {
	const {
		version_name,
		update_content,
		download_url,
		is_force,
		version_code
	} = versionInfo

	uni.showModal({
		title: `发现新版本 v${version_name}`,
		content: update_content || '程序有新版本，建议更新',
		showCancel: !Boolean(is_force),
		confirmText: '立即更新',
		success(res) {
			if (res.confirm) {
				downloadAndInstallWgt(download_url, version_code)
			} else if (is_force) {
				showWgtUpdateModal(versionInfo)
			}
		}
	})
}

/**
 * 下载并安装WGT热更包
 * @param {String} wgtUrl wgt文件下载地址
 * @param {Number} newVer 本次热更的版本号
 */
function downloadAndInstallWgt(wgtUrl, newVer) {
	const task = uni.downloadFile({
		url: wgtUrl,
		success: (res) => {
			uni.hideLoading()
			if (res.statusCode !== 200) {
				return uni.showModal({
					title: '更新失败',
					content: '资源包下载异常，请稍后重试',
					showCancel: false
				})
			}
			plus.runtime.install(
				res.tempFilePath,
				{
					force: false
				},
				() => {
					// 安装成功：写入本地缓存，永久记录当前最高热更版本
					uni.setStorageSync('wgt_update_ver', newVer)
					uni.showModal({
						title: '更新完成',
						content: '应用即将重启',
						showCancel: false,
						success: () => {
							plus.runtime.restart()
						}
					})
				},
				(err) => {
					uni.showModal({
						title: '更新失败',
						content: `安装失败：${err.message}`,
						showCancel: false
					})
				}
			)
		},
		fail: () => {
			uni.hideLoading()
			uni.showModal({
				title: '更新失败',
				content: '网络异常，下载中断',
				showCancel: false
			})
		}
	})

	task.onProgressUpdate((progress) => {
		uni.showLoading({
			title: `正在下载 ${progress.progress}%`
		})
	})
}

/**
 * 整包更新弹窗
 */
function showUpdateModal(versionInfo) {
	const {
		version_name,
		update_content,
		download_url,
		is_force
	} = versionInfo

	uni.showModal({
		title: `发现新版本 v${version_name}`,
		content: update_content || '发现新版本，请及时更新',
		showCancel: !Boolean(is_force),
		confirmText: '立即更新',
		success(res) {
			if (res.confirm) {
				plus.runtime.openURL(download_url)
			} else if (is_force) {
				showUpdateModal(versionInfo)
			}
		}
	})
}
```

##  请求接口类封装

### 统一api接口请求  `util\request\request.js`

```js
// 获取环境标识
const env = import.meta.env.VITE_APP_ENV

// 定义 baseUrl（在外面定义，if 里只赋值）
let baseUrl

// 判断环境
if (env === 'local') {
	// 本地环境
	baseUrl = 'http://www.cs.com/api'
} else {
	// 生产环境
	 baseUrl = 'http://47.107.33.56:1997/api'
	//baseUrl = 'http://www.cs.com/api'
}

export default function http(url, data = {}, method = 'GET') {
	return new Promise((resolve, reject) => {
		// 获取token
		const token = uni.getStorageSync('token') || ''

		// 初始化请求头
		const header = {}

		// 核心修改：token 不为空时，才添加到请求头
		if (token) {
			header.token = 'Bearer ' + token
			header.Authorization = 'Bearer ' + token
		}

		uni.request({
			url: baseUrl + url,
			data,
			method: method.toUpperCase(), // 统一大写
			header: header, // 使用处理后的请求头
			success: (res) => {
				// 1. HTTP状态码正常
				if (res.statusCode === 200) {
					const resData = res.data

					// 业务成功
					if (resData.code === 0) {
						resolve(resData.data)
					}
					// 业务失败
					else if (resData.code === 1000) {
						uni.showToast({
							title: resData.msg || '请求失败',
							icon: 'none'
						})
						reject(resData.msg)
					}
					// 其他业务码
					else {
						uni.showToast({
							title: '数据格式异常',
							icon: 'none'
						})
						reject('数据格式异常')
					}
				}
				// 2. HTTP状态码异常（401/404/500等）
				else {
					uni.showToast({
						title: `请求错误 ${res.statusCode}`,
						icon: 'none'
					})
					reject(res)
				}
			},
			fail: (err) => {
				// 网络错误/超时
				uni.showToast({
					title: '网络异常，请稍后重试',
					icon: 'none'
				})
				reject(err)
			}
		})
	})
}
```
