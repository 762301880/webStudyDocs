### 条件编译

条件编译是用特殊的注释作为标记，在编译时根据这些特殊的注释，将注释里面的代码编译到不同平台。

### 使用方法

以 `#ifdef` 或 `#ifndef` 加 `%PLATFORM%` 开头，以 `#endif` 结尾。

- `#ifdef`：if defined 仅在某平台存在
- `#ifndef`：if not defined 除了某平台均存在
- `%PLATFORM%`：平台名称

### 全局多端判断封装方案

#### 工具函数（推荐，全局导入使用）

新建文件 `src/utils/platform.js`

```js
/**
 * 获取当前运行端标识
 * @returns {string} app / h5 / wechat / alipay / douyin / qq / kuaishou / baidu
 */
export function getPlatform() {
  // APP端
  // #ifdef APP-PLUS
  return 'app'
  // #endif

  // H5网页
  // #ifdef H5
  return 'h5'
  // #endif

  // 微信小程序
  // #ifdef MP-WEIXIN
  return 'wechat'
  // #endif

  // 支付宝小程序
  // #ifdef MP-ALIPAY
  return 'alipay'
  // #endif

  // 抖音/头条小程序
  // #ifdef MP-TOUTIAO
  return 'douyin'
  // #endif

  // QQ小程序
  // #ifdef MP-QQ
  return 'qq'
  // #endif

  // 快手小程序
  // #ifdef MP-KUAISHOU
  return 'kuaishou'
  // #endif

  // 百度小程序
  // #ifdef MP-BAIDU
  return 'baidu'
  // #endif
}

/**
 * 获取传给后端的source来源字段
 * @returns {string} APP / H5 / WECHAT / ALIPAY / DOUYIN
 */
export function getSource() {
  // #ifdef APP-PLUS
  return 'APP'
  // #endif
  // #ifdef H5
  return 'H5'
  // #endif
  // #ifdef MP-WEIXIN
  return 'WECHAT'
  // #endif
  // #ifdef MP-ALIPAY
  return 'ALIPAY'
  // #endif
  // #ifdef MP-TOUTIAO
  return 'DOUYIN'
  // #endif
  return 'OTHER'
}

/**
 * 平台判断快捷方法
 */
export const platform = {
  isApp() {
    // #ifdef APP-PLUS
    return true
    // #endif
    return false
  },
  isH5() {
    // #ifdef H5
    return true
    // #endif
    return false
  },
  isWechat() {
    // #ifdef MP-WEIXIN
    return true
    // #endif
    return false
  },
  isAlipay() {
    // #ifdef MP-ALIPAY
    return true
    // #endif
    return false
  },
  isDouyin() {
    // #ifdef MP-TOUTIAO
    return true
    // #endif
    return false
  },
  // 是否任意小程序
  isMp() {
    // #ifdef MP
    return true
    // #endif
    return false
  }
}
```

#### **全局挂载 main.js（页面无需每次 import）**

```js
import { getPlatform, getSource, platform } from '@/utils/platform'

// 挂载到全局
uni.$getPlatform = getPlatform
uni.$getSource = getSource
uni.$platform = platform
```

#### **页面 / 组件中使用示例**

```js
export default {
  onLoad() {
    // 获取端标识
    const p = uni.$getPlatform()
    console.log('当前端', p)

    // 登录接口使用source
    const source = uni.$getSource()
    this.login(source)

    // 快捷布尔判断
    if (uni.$platform.isApp()) {
      console.log('APP端逻辑')
    }
    if (uni.$platform.isWechat()) {
      console.log('微信小程序逻辑')
    }
    if (uni.$platform.isMp()) {
      console.log('所有小程序通用逻辑')
    }
  },
  methods: {
    login(source) {
      console.log('提交来源', source)
      // api.login({ source })
    }
  }
}
```

#### 补充说明

1. 编译期无多余代码

   条件编译打包时只会保留当前平台的 return，其他分支直接删除，无运行时判断损耗；

2. 如果你想用在模板 template

   可以在页面 `computed`中映射全局方法：

```js
computed: {
  isApp() {
    return uni.$platform.isApp()
  }
}
```

```vue
<view v-if="isApp">仅APP展示</view>
```

运行时兼容兜底（可选扩展）

如果需要动态判断（不推荐，仅特殊场景），可以搭配 `uni.getSystemInfoSync` 做双重校验。

## 版本控制

### sql表设计

```sql
CREATE TABLE `fa_app_version` (
  `id` int NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `app_id` varchar(100) NOT NULL COMMENT '项目manifest中的appid，用于多项目隔离',
  `platform` tinyint NOT NULL COMMENT '平台类型：1-安卓，2-iOS',
  `version_name` varchar(20) NOT NULL COMMENT '版本展示号，格式如1.3.2',
  `version_code` int NOT NULL COMMENT '数字版本号，用于版本大小比对，每次打包自增',
  `wgt_version` varchar(20) DEFAULT NULL COMMENT '热更新wgt包版本号',
  `update_type` tinyint NOT NULL COMMENT '更新类型：1-热更新，2-整包更新',
  `is_force` tinyint DEFAULT '0' COMMENT '是否强制更新：0-非强制，1-强制更新',
  `min_support_code` int DEFAULT '0' COMMENT '最低兼容版本code，低于该版本强制整包升级',
  `download_url` varchar(500) DEFAULT NULL COMMENT '安装包/apk/ipa/wgt下载地址',
  `update_content` text COMMENT '更新日志、新版本说明',
  `status` tinyint DEFAULT '1' COMMENT '版本状态：0-下线，1-线上生效',
  `create_time` datetime DEFAULT NULL COMMENT '版本创建时间',
  PRIMARY KEY (`id`)
) ENGINE=InnoDB AUTO_INCREMENT=3 DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_0900_ai_ci COMMENT='APP版本更新管理表';
```

### 相关字段的解释

| 字段         | 作用                 | 示例             |
| ------------ | -------------------- | ---------------- |
| app_id       | 标识是哪个App        | `__UNI__1234567` |
| version_name | 给用户看的版本号     | `3.0.1`          |
| version_code | 程序比较大小的版本号 | `111`            |

#### app_id

用于识别应用身份。

相当于：

```shell
身份证号
```

例如：

```shell
用户端App
商家端App
管理端App
```

数据库：

```shell
app_id=__UNI__USER
app_id=__UNI__SHOP
app_id=__UNI__ADMIN
```

UniApp获取：

```javascript
console.log(plus.runtime.appid)
```

例如：

```php
__UNI__F4A1234
```

这个值基本不会变。

#### version_name

用户看到的版本号。

例如：

```bash
我的 -> 关于我们

当前版本：
3.0.1
```

App Store：

```bash
Version 3.0.1
```

Android：

```bash
版本 3.0.1
```

都是这个。

升级过程：

```bash
3.0.0
↓
3.0.1
↓
3.0.2
↓
3.1.0
```

用户能看懂。

#### version_code(版本控制比较用这个字段)

程序自己比较大小。

例如：

```bash
3.0.1
```

对应：

```bash
111
```

下一版：

```bash
3.0.2
```

对应：

```bash
112
```

再下一版：

```bash
3.1.0
```

对应：

```bash
120
```

因为程序不能直接这样比较：

```bash
"3.0.10" > "3.0.2"
```

结果可能不符合预期。

所以一般比较：

```bash
111 < 112
```

最简单。



#### 这个表里怎么用

假设：

```bash
当前安装版本
3.0.0
version_code=110
```

服务器返回：

```json
{
    "app_id":"__UNI__F4A1234",
    "version_name":"3.0.1",
    "version_code":111
}
```

检测：

```javascript
if(server.version_code > local.version_code){
    // 需要升级
}
```

#### 实际生产环境推荐

数据库：

```bash
app_id='__UNI__F4A1234'

version_name='3.0.1'

version_code=111
```

显示给用户：

```bash
当前版本：3.0.1
```

代码判断：

```bash
111 > 110
```



#### 如果只有一个App

甚至可以理解成：

```bash
app_id
    ↓
哪个App

version_name
    ↓
给人看的

version_code
    ↓
给程序比较的
```

最简单的例子：

```bash
app_id       = __UNI__F4A1234

version_name = 3.0.1

version_code = 111
```

以后发新版：

```bash
app_id       = __UNI__F4A1234  （不变）

version_name = 3.0.2

version_code = 112
```

真正用于“检测是否需要更新”的核心字段其实是：

```bash
version_code
```

程序只需要比较：

```bash
服务器version_code > 本地version_code
```

就知道是否有新版本。

###  **版本判断最好放在 `App.vue` 的 `onLaunch()` 里。**

原因：

- App启动立即检查
- 所有页面进入前就能拦截
- 强制更新可以直接阻止用户继续使用
- 绝大多数项目都是这么做

```js
// App.vue
export default {
    onLaunch() {
        this.checkVersion()
    }
}
```

如果你担心每次启动都请求接口，可以：

```js
App启动
↓
检查版本
↓
有新版本弹窗
↓
进入首页
```

###  具体业务逻辑 util/version.js

```js
import {
	request
} from "@/utils/request.js"
/**
 * 检查版本更新
 */
export async function checkVersion() {
	// #ifdef H5
	return
	// #endif

	try {

		const source = uni.getStorageSync('source')
		const fn = async () => {
			// 重点：确认是 this.request 还是全局 request
			const res = await request('app_version/appVersion', {}, 'GET')
			const {
				code,
				msg,
				data
			} = res
			if (code !== 1 || !data) {
				return
			}
			const serverVersion = Number(res.data.version_code)
			const localVersion = Number(plus.runtime.versionCode || 0)

          console.log(serverVersion,localVersion)

			// 当前已是最新版本
			if (localVersion >= serverVersion) {
				return
			}
			
			showUpdateModal(res.data)
		}
		fn()
	} catch (error) {
		console.error('检查更新失败', error)
	}
}

/**
 * 更新提示
 */
function showUpdateModal(versionInfo) {
	const {
		version_name,
		update_content,
		download_url,
		is_force
	} = versionInfo

	uni.showModal({
		title: `发现新版本 v${version_name}`,
		content: update_content || '发现新版本，请及时更新',
		showCancel: !Boolean(is_force),
		confirmText: '立即更新',

		success(res) {
			if (res.confirm) {
				plus.runtime.openURL(download_url)
			} else if (is_force) {
				// 强制更新，继续弹
				showUpdateModal(versionInfo)
			}
		}
	})
}
```



## 封装打包后的终端来源

### utils/source.js

```js
/**
 * 获取当前终端来源
 */
export function getSource() {
  // APP环境统一先匹配
  // #ifdef APP-PLUS
  const os = plus.os.name.toLowerCase()
  if (os === 'android') return 'android'
  if (os === 'ios') return 'ios'
  // #endif

  // #ifdef H5
  return 'h5'
  // #endif

  return 'unknown'
}

/**
 * 初始化来源并存储
 */
export function initSource() {
  const source = getSource()
  console.log(source, '222222222222222222222222')
  uni.setStorageSync('source', source)
  return source
}
```

### App.vue

```js
import { initSource } from '@/utils/source.js'

export default {
  onLaunch() {
    initSource()
  }
}
```

### 使用

```js
import { getSource } from '@/utils/source.js'

```
