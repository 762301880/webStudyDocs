### 自定义调试基座是什么（通俗讲）

#### 一、先分清两个概念

1. 标准调试基座（官方默认基座）

   HBuilderX 自带的通用安卓调试包，没有你的项目专属配置，所有 uniapp 项目共用这一个安装包。

   缺点：

- 不支持你项目配置的第三方 SDK（微信登录、支付、推送、地图、极光、个推等）
- 无法测试自定义包名、签名、应用权限、原生插件
- 真机调试第三方功能会直接报错

2. 自定义调试基座

​    用你当前项目的原生配置、插件、包名、签名，单独编译出来的专属调试安装包，只给你这个项目用。

​    简单一句话：把你项目的原生环境打包成一个调试用 APP，替代官方通用基座。

####  二、什么时候必须用自定义调试基座？

只要你的项目满足任意一条，就必须打包自定义基座才能真机调试：

1. 使用了原生插件（市场插件、自有原生插件）
2. 集成第三方 SDK：微信 / 支付宝登录支付、腾讯地图、百度定位、推送 SDK、人脸识别等
3. 修改了应用包名、自定义签名证书
4. 配置了 AndroidManifest 自定义权限、Activity、Scheme 跳转
5. 需要测试 APP 本地通知、分享、拉起第三方应用

只用纯 uniapp 页面、无任何原生插件 / 第三方 SDK，可以不用自定义基座，直接用官方基座调试。

#### 三、自定义调试基座 和 正式打包的区别

| 对比项   | 自定义调试基座                            | 正式发行包（打包 APK）       |
| -------- | ----------------------------------------- | ---------------------------- |
| 用途     | 开发阶段真机调试、联调 SDK / 插件         | 上线分发给用户安装           |
| 编译速度 | 快，只编译调试环境                        | 慢，会混淆、压缩、校验签名   |
| 调试能力 | 支持 HBuilderX 热更新、断点调试、打印日志 | 关闭调试，不支持热重载       |
| 签名     | 可使用测试证书                            | 必须正式证书（上架商店要求） |
| 体积     | 偏大，包含调试工具                        | 精简，去除调试代码           |

#### 四、完整使用流程

1. HBuilderX 打开项目 → 菜单：**发行 → 原生 App - 制作自定义调试基座**
2. 配置安卓签名、包名、勾选需要的原生插件
3. 等待编译完成，生成专属调试 APK
4. 手机安装这个基座，连接 HBuilderX 真机运行
5. 之后点「运行到手机 / 模拟器」，会自动把前端代码跑在这个自定义基座里，所有 SDK、原生功能都能正常调试

####  五、常见误区

1. 误区：自定义基座 = 正式安装包

   错，基座只是调试工具，不能发给用户上架，上架要走「原生 App - 云打包」发行包。

2. 误区：每次改插件 / 包名都要重新生成基座

   对，只要修改原生配置（插件、包名、权限、签名），必须重新制作自定义基座，否则配置不生效。

3. 误区：不用自定义基座也能测微信登录

   不行，微信开放平台需要绑定对应包名和签名，官方通用基座包名固定，无法匹配你后台配置，登录直接失败。

#### 六、总结

自定义调试基座 = **带本项目全部原生配置的调试专用 APP**，核心作用是让你在开发阶段，真机完整测试所有原生插件、第三方 SDK、原生权限配置，解决官方通用基座不支持原生能力的问题。

### app真机运行

点击导航栏的**运行>运行到手机或模拟器>运行到Android App基座**  会提醒你安装app真机环境
