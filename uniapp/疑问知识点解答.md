## unapp为什么不推荐写div

### 一、为什么不建议用 div（核心原因）

uni-app 是**跨平台框架**，要同时跑在：

- ✅ H5（浏览器，支持 div）
- ⚠️ 微信 / 支付宝 / 抖音小程序（**不支持 HTML 原生标签**，自有组件体系）
- ⚠️ App（iOS/Android，webview 渲染，兼容有限）

#### 1. 小程序端根本不认 div

小程序的视图层是 **WXML**，不是 HTML：

- 小程序里没有 `<div>`、`<span>`、`<p>`
- 只能用它内置的组件：**view、text、image、navigator** 等

你在 uni-app 写：

```html
<div>内容</div>
```

编译到微信小程序后，会变成**不认识的标签**，直接原样显示或不渲染，样式也完全失效。

#### 2. 编译机制：div 不会转成 view

uni-app 编译器**不会自动把 div 转成 view**：

- H5：div → 正常 div
- 小程序 / App：div → 保留为 `<div>`，但平台不识别 → 出错

而你写：

```html
<view>内容</view>
```

编译器会自动适配：

- H5：view → 编译成 div
- 小程序：view → 原生 view 组件
- App：view → 对应原生容器

#### 3. 样式选择器限制

非 H5 端（小程序 / App）不支持：

```css
div { ... } /* ❌ 无效 */
```

因为 WXSS 不允许用 HTML 标签选择器，只能用 **class、id、小程序组件名**。

------

### 二、实际表现（你会遇到的问题）

1. **小程序预览：文字正常，但布局错乱、样式不生效**
2. **App 端：可能直接不显示、空白或排版混乱**
3. **控制台警告：invalid tag name 'div'**

------

### 三、正确写法：用 view 替代 div

#### 1. 替换规则（HTML → uni-app）

|   HTML   | uni-app 推荐  |        说明         |
| :------: | :-----------: | :-----------------: |
| `<div>`  |   `<view>`    |  块级容器，最常用   |
| `<span>` |   `<text>`    | 行内文本，必用 text |
|  `<p>`   |   `<view>`    | 段落用 view 包 text |
| `<img>`  |   `<image>`   |      图片组件       |
|  `<a>`   | `<navigator>` |      跳转链接       |

#### 2. 示例

❌ 错误（H5 正常，小程序 / App 炸）：

```html
<template>
  <div class="box">
    <span>文字</span>
  </div>
</template>
```

✅ 正确（全平台兼容）：

```html
<template>
  <view class="box">
    <text>文字</text>
  </view>
</template>
```

------

### 四、特殊情况：非要用 div 怎么办？

只有**纯 H5 项目**可以随便用 div；一旦要打包小程序 / App，**必须全换成 view**。

如果是富文本内容（后端返回 HTML）：

- 用 `<rich-text>` 组件渲染，不要直接写 div

------

### 五、总结

- **不是不能用，是跨平台会崩**：div 是 HTML 标签，小程序 / App 不支持
- **官方替代：view**：全平台自动适配，写法几乎一样
- **一句话记牢**：uni-app 里，**块级用 view、行内用 text、图片用 image**
